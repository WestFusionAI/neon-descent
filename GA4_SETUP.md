# Google アナリティクス（GA4）導入手順（無料・GitHub Pages）

名刺のQRコードは **URLが変わらなければ再印刷不要** です。  
例: `https://westfusionai.github.io/neon-descent/?from=card`

---

## 1. GA4 でプロパティを作る

1. ブラウザで [Google アナリティクス](https://analytics.google.com/) を開き、Googleアカウントでログインします。
2. **管理**（歯車）→ **アカウントを作成**（初回のみ）→ **プロパティを作成** を進めます。
3. **プロパティ名**（例: `NEON DESCENT`）やタイムゾーン・通貨を設定します。
4. **ビジネスの詳細**・**ビジネス目標**は任意で進めます（スキップ可能な項目もあります）。

---

## 2. データストリーム（Web）を登録する

1. **データの収集** → **データストリーム** → **ストリームを追加** → **ウェブ** を選びます。
2. **ウェブサイトのURL** に次を入力します（末尾スラッシュの有無はどちらでも可なことが多いです）。

   `https://westfusionai.github.io/neon-descent/`

3. **ストリーム名**（例: `neon-descent GitHub Pages`）を付けて保存します。
4. 表示される **測定ID**（形式: `G-XXXXXXXXXX`）をメモします。  
   ※このIDは後で `index.html` に埋め込みます。

---

## 3. サイトにタグを埋め込む（このリポジトリ）

1. ローカルで `index.html` を開き、`<head>` 内の **できるだけ上の方**（`<meta charset>` の直後あたり）に、次のスクリプトを追加します。
2. **`G-XXXXXXXXXX` を、手順2でメモした自分の測定IDに置き換えます。**

```html
<!-- Google tag (gtag.js) -->
<script async src="https://www.googletagmanager.com/gtag/js?id=G-XXXXXXXXXX"></script>
<script>
  window.dataLayer = window.dataLayer || [];
  function gtag(){dataLayer.push(arguments);}
  gtag("js", new Date());
  gtag("config", "G-XXXXXXXXXX");
</script>
```

3. 変更をコミットし、`master`（またはデプロイ用ブランチ）へ push します。
4. GitHub Pages の反映を数分待ちます。

---

## 4. データが入っているか確認する

1. GA4 の **レポート** → **リアルタイム** を開きます。
2. スマホまたはPCでゲームURLを開き、**アクティブユーザー**や**イベント**が増えるか確認します。
3. 数時間〜1日程度で、**レポート**の各画面に通常の集計が溜まり始めます。

---

## 5. 「名刺（`from=card`）」経由だけを見たい場合

GA4 は **ページのURL（クエリ付き）** を記録します。

- **レポート** → **エンゲージメント** → **ページとスクリーン** などで、URL に `from=card` が含まれる行を探す。
- または **探索** で「ページパス＋クエリ」やディメンションを使ってフィルタする（慣れが必要な場合あり）。

より分かりやすく分けたい場合は、別途 **カスタムディメンション** や **イベント送信**（`gtag("event", ...)`）を追加する方法もあります（上級）。

---

## 6. よくある注意

- **地域**や**回数**は **IP等に基づく推定** で、VPN・モバイル回線・会社プロキシでは実際と異なることがあります。
- **広告ブロッカー**を使っているユーザーは計測されないことがあります。
- 利用者がEU圏などに広がる場合は、**Cookie同意バナー**等の要件を別途確認してください（日本国内のみの想定でも、方針を決めておくと安全です）。

---

## 7. 名刺のQRを変える必要があるケース（参考）

次のときだけ、QRの再発行が必要です。

- ドメインやパスを変えた（別URLに移した）
- 名刺用の入口URLを別のものにしたい（短縮URLに切り替える等）

**GA4を追加しただけ**ではURLは変わらないので、**QRはそのままで問題ありません。**

---

## 8. クリア率など（カスタムイベント）

`index.html` から次のイベントが送られます（`gtag` がブロックされていなければ）。

| イベント名 | タイミング |
|------------|------------|
| `nd_game_start` | START で本番プレイが始まったとき（分母に使える） |
| `nd_game_over` | ゲームオーバーになったとき |
| `nd_all_clear` | 全ステージクリア（本記録のみ。`?demo=ending` のデモでは送らない） |

主なパラメータ: `from_card`（`yes` / `no`）、`demo_query`（URL にエンディングデモ用クエリがあるとき `yes`）、`enemy_bullet_tier`、`final_score`。

**クリア率の見方（例）:** GA4 の **探索** → **自由形式** などで、イベント名 `nd_game_start` のイベント数を分母、`nd_all_clear` を分子として比較する。本番のみに絞るときは `demo_query` が `no` の行だけをフィルタする。

イベントパラメータをディメンションとして常に使う場合は、GA4 **管理** → **カスタム定義** で各パラメータを登録する（登録後、データが溜まり始めてから探索に出る）。
