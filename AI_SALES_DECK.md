# AI活用デモ資料（提案・活動記録）

## 概要
- **デモ**: ブラウザで動く2Dシューティングゲーム「NEON DESCENT」
- **目的**: 「AIを使って短期間でプロトタイプを作り、要件追加・調整・公開まで回す」実例として提示する
- **公開URL（GitHub Pages / PWA対応）**: `https://westfusionai.github.io/neon-descent/`
- **リポジトリ**: `https://github.com/WestFusionAI/neon-descent`

## 何が“営業に効く”ポイントか（AI活用の見せどころ）
- **仕様追加の反復が速い**: 「敵が弾を撃つ」「横から出る敵」「ボス弾を優しく」「スマホ操作UI」「感度調整」「公開・更新」などを段階的に実装
- **ユーザー体感を中心に調整**: 難易度（敵弾の有無/強さ）や操作感度をUIで調整可能にし、フィードバックに合わせてチューニング
- **“作って終わり”ではなく配布まで**: GitHub Pagesで公開、PWA（ホーム画面追加）でアプリ風運用、更新反映（SW更新戦略）まで整備

## ゲーム説明（NEON DESCENT）
### 基本ルール
- 自機を操作して敵を撃破しスコア獲得
- アイテム取得で一時的な強化（連射/拡散/パワー/シールド/ライフ/オプション）
- ステージ進行で中ボス・大ボスが出現（ステージ進行に応じて難易度が上がる設計）

### 実装済み機能（動作一覧）
- **敵の種類**
  - 上から降ってくる通常敵（複数タイプ）
  - 画面左右から侵入する敵（横入りタイプ）
  - 中ボス・大ボス（定期射撃あり）
- **攻撃**
  - プレイヤー弾（通常＋強化時3-wayなど）
  - **敵弾**（雑魚/ボス）  
    - タイトルで「敵の弾：なし/標準/多め」を選択可能
- **被弾・残機・シールド**
  - 体当たり/敵弾の被弾判定
  - 無敵時間、シールド消費、残機処理、ゲームオーバー表示
- **スコア/コンボ**
  - コンボ加算、表示更新、撃破に応じた得点
- **演出・サウンド**
  - BGM（簡易シーケンス）
  - SE（射撃/爆発/被弾/敵射撃など）
  - 背景（宇宙）＋「宇宙ステーション/他船/隕石」などの“場面”演出
- **スマホ操作（両手操作想定）**
  - 左下：アナログスティック（中心を基準に倒す）
  - 右下：FIRE（長押し連射）、OP砲、⏸
  - タイトル/ポーズ/結果画面では操作UIを非表示（誤タップ競合を回避）
- **操作感度調整**
  - タイトル画面：操作感度（移動スピード）スライダー
  - ポーズ画面：プレイ中も感度調整→即反映
  - 設定値は保存され次回起動時に復元
- **終了（EXIT）**
  - ポーズ画面/結果画面に「終了（EXIT）」ボタン
  - タイトルに戻し、タブを閉じる案内（※モバイルChrome仕様で強制クローズは不可）

## 操作方法
### PC
- **移動**: WASD / 矢印
- **メイン射撃**: SPACE（長押し）
- **オプション砲**: J / X（取得後）
- **一時停止**: ⏸ ボタン / ESC / P

### スマホ（Android Chrome / iPhone Safari）
- **移動**: 左下スティック（倒した方向・量に追従）
- **射撃**: 右下 FIRE（長押し連射）
- **オプション砲**: OP（取得後）
- **一時停止**: ⏸

## どこでも使える形（公開・更新・PWA）
### 公開（GitHub Pages）
- GitHub PagesでHTTPS配信
- URLを共有するだけで誰でもブラウザで起動可能

### “アプリっぽく”使う（PWA）
- Android Chrome: 「ホーム画面に追加」/「アプリをインストール」
- iPhone Safari: 共有→「ホーム画面に追加」
- Service Workerにより、起動や更新の体験を安定化

### 更新運用（随時更新）
- ローカルで更新 → `git commit` → `git push`
- Pages側が自動で再ビルド/反映
- PWAはキャッシュが絡むため、更新戦略（SW更新）も実装済み

### 3) 最終的に“見せられる形”へ
